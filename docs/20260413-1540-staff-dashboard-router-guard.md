# 当日用管理画面 ルーターガード（Turn 3）

## 目的

`/staff` に未ログインでアクセスすると白画面になる問題を解消する。
Django admin のログイン画面にリダイレクトし、ログイン後に `/staff` に戻る導線を作る。
また、操作中にセッションが切れた場合も同様にリダイレクトする。

## 変更ファイル一覧

| # | ファイル | 変更内容 |
|---|---------|---------|
| 1 | `frontend/src/router/index.js` | `beforeEach` ガードを追加。`requiresAuth` なルートで認証チェック → 未認証なら `/admin/login/` へリダイレクト |
| 2 | `frontend/src/api/client.js` | レスポンスインターセプターを追加。staff ページで 401/403 を受けたらログイン画面へリダイレクト |

## 各ファイルの変更内容

### router/index.js
- `client` をインポート
- `router.beforeEach` を追加
- `meta.requiresAuth` が true のルートのみ認証チェックを実行
- `GET /api/staff/reservations/?search=__auth_check__` を叩いて認証状態を確認（専用エンドポイント不要）
- 401/403 → `window.location.href = '/admin/login/?next=/staff'` でリダイレクト
- それ以外のエラー（ネットワーク障害等）→ 通過させる（画面内で個別にエラー表示）

### api/client.js
- レスポンスインターセプターを追加
- 401/403 かつ `window.location.pathname` が `/staff` で始まる場合のみリダイレクト
- 公開ページ（予約確認等）の API コールには影響しない

## バックエンド変更

なし。既存の `GET /api/staff/reservations/` の IsAuthenticated を認証チェックとして流用。

## 設計判断

- **独自ログイン画面は作らない**: Django admin のログイン画面を流用。`?next=` パラメータでログイン後に `/staff` に戻る
- **認証チェックに既存APIを流用**: `__auth_check__` という存在しない検索語で軽量リクエスト。新規エンドポイント不要
- **インターセプターは `/staff` ページのみ**: 公開ページの API コール（予約確認、チェックイン等）でリダイレクトが発生しない

```bash
cd frontend && npx vite build
```

## 動作確認結果

- ビルド成功（エラーなし）

## 本番での動作確認手順

1. 未ログイン状態で `/staff` にアクセス → Django admin ログイン画面にリダイレクトされること
2. ログイン後 → `/staff` に戻ること
3. ログイン済みで `/staff` にアクセス → 通常通り受付画面が表示されること
4. セッション切れ状態で操作（検索、現金受領等） → ログイン画面にリダイレクトされること
5. 公開ページ（`/`, `/:slug`, `/reservation/:token`）は認証不要で表示されること
